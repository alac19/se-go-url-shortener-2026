## 三、代码规范

### 3.1 命名规范

| 类型       | 规则                           | 示例                          |
| ---------- | ------------------------------ | ----------------------------- |
| 包名       | 小写、单数、无下划线           | `handler`, `service`          |
| 结构体     | 大驼峰                         | `ShortLinkService`            |
| 接口       | 大驼峰，单方法可加 `-er`       | `LinkRepository`              |
| 函数/方法  | 大驼峰（导出）或小驼峰（私有） | `GenerateShortLink`, `newRepo`|
| 变量       | 大驼峰（导出）或小驼峰（私有）                         | `shortCode`, `LongURL`        |
| 常量       | 大驼峰或全大写                 | `MaxRetries`                  |
| 文件名     | 小写，下划线分隔               | `shortlink_test.go`           |

### 3.2 注释规范

- 每个导出的符号必须有文档注释，以符号名开头。
- 示例：
  ```go
  // GenerateShortLink 创建短链接，返回短码。
  // 参数：
  //   - ctx: 上下文
  //   - longURL: 原始长链接
  // 返回：
  //   - 短码（不含域名）
  //   - 错误信息
  func (s *ShortLinkService) GenerateShortLink(ctx context.Context, longURL string) (string, error) { ... }
  ```
- 包注释放在包声明上方，说明包的作用。

### 3.3 错误处理

- 使用 `fmt.Errorf` 或 `errors.New` 创建错误。
- 错误信息小写开头，不加标点。
- 在 service/repository 层向上返回明确错误（如 `sql.ErrNoRows` 转换为自定义错误）。
- 在 handler 层根据错误类型设置 HTTP 状态码。

### 3.4 代码风格

- 使用 `go fmt` 自动格式化。
- 每行长度不超过 120 字符（建议）。
- 禁止使用 `panic`（除初始化失败外）。
- 导出函数按逻辑分组，使用空行分隔。

## 四、配置规范

### 4.1 配置文件格式

使用 TOML，示例 `configs/config.toml`：

```toml
[server]
port = 8080
mode = "debug"   # debug / release

[mysql]
dsn = "root:123456@tcp(localhost:3306)/shortlink_db?charset=utf8mb4&parseTime=True&loc=Local"

[redis]
addr = "localhost:6379"
password = ""
db = 0

[rate_limit]
requests_per_minute = 5   # 每个 IP 每分钟最多生成短链次数
```

### 4.2 配置加载

- 优先从环境变量读取（便于容器化），否则从 `config.toml` 读取。

## 五、日志规范

- **日志库**：`log/slog`（Go 1.25+）。
- **输出格式**：JSON（生产环境）或 Text（开发环境）。
- **日志级别**：
  - `DEBUG`：详细调试信息（开发）
  - `INFO`：正常请求日志（方法、路径、短码、耗时）
  - `WARN`：限流触发、缓存降级
  - `ERROR`：数据库/Redis 连接失败、内部错误
- **示例日志行**：
  ```json
  {"time":"2026-06-04T10:00:00Z","level":"INFO","msg":"GET /abc123 302 2ms","short_code":"abc123","status":302}
  ```

## 六、测试规范

### 6.1 单元测试

- 每个 `_test.go` 文件使用表驱动测试。
- 使用 `go-sqlmock` 模拟 MySQL 操作，`miniredis` 模拟 Redis。
- 覆盖率目标：≥ 60%。

### 6.2 集成测试

- 使用 `testcontainers-go` 启动真实 MySQL/Redis 容器，测试完整流程。
- 或使用 `docker-compose` 启动依赖，然后运行测试。

### 6.3 基准测试

- 为关键函数编写 `BenchmarkXXX`，如 `BenchmarkBase62Encode`、`BenchmarkRedisGet`。

### 6.4 压测

- 使用 `go-wrk` 或 `ab` 对重定向接口进行压测，记录 QPS 和延迟。

## 七、构建与部署规范

### 7.1 本地构建

```bash
go build -o shortlink-server ./cmd/server
```

### 7.2 Docker 镜像

多阶段 Dockerfile 示例：

```dockerfile
FROM golang:1.25-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -o shortlink-server ./cmd/server

FROM alpine:latest
RUN apk --no-cache add ca-certificates
WORKDIR /root/
COPY --from=builder /app/shortlink-server .
COPY configs/config.toml ./configs/
EXPOSE 8080
CMD ["./shortlink-server"]
```

- 目标镜像大小 ≤ 30MB。

### 7.3 Docker Compose

`docker-compose.yml` 示例：

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: shortlink_db
    ports:
      - "3306:3306"
  redis:
    image: redis:7.2
    ports:
      - "6379:6379"
  shortlink:
    build: .
    ports:
      - "8080:8080"
    depends_on:
      - mysql
      - redis
    environment:
      MYSQL_DSN: "root:123456@tcp(mysql:3306)/shortlink_db?charset=utf8mb4&parseTime=True&loc=Local"
      REDIS_ADDR: "redis:6379"
```

### 7.4 版本管理

- 使用 `git tag` 标记版本，如 `v1.0.0`。

- **main**：稳定版，只接受 `develop` 的 PR。
