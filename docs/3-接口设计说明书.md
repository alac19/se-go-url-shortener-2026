## 一、引言

### 1.1 文档目的
本文档定义短链接服务对外提供的 HTTP API 接口，包括请求方法、路径、参数、响应格式及错误码，供前端或第三方调用方参考。

### 1.2 适用范围
本文档适用于短链接服务的开发、测试及对接阶段。响应数据格式为 JSON。

## 二、外部接口

### 2.1 生成短链接

**接口描述**：将长链接转换为短链接。

- **请求方法**：`POST`
- **请求路径**：`/api/links`
- **请求头**：`Content-Type: application/json`
- **请求体**：
  ```json
  {
    "url": "https://example.com/very/long/url"
  }
  ```
- **响应成功（200 OK）**：
  ```json
  {
    "short_url": "http://short.xxx/abc123"
  }
  ```
- **错误响应**：
  - `400 Bad Request`：URL 无效或格式错误
  - `429 Too Many Requests`：触发限流（每分钟生成次数超限）
  - `500 Internal Server Error`：服务内部错误

### 2.2 重定向

**接口描述**：根据短码跳转到原始长链接。

- **请求方法**：`GET`
- **请求路径**：`/{code}`（例如 `/abc123`）
- **请求参数**：无
- **响应**：
  - 成功：`HTTP 302 Found`，`Location` 头为原始长链接
  - 失败：
    - `404 Not Found`：短码不存在
    - `500 Internal Server Error`：服务内部错误

### 2.3 查询短链统计信息（可选）

**接口描述**：获取某个短链的点击次数等统计信息。

- **请求方法**：`GET`
- **请求路径**：`/api/stats/{code}`
- **响应成功（200 OK）**：
  ```json
  {
    "short_code": "abc123",
    "click_count": 42,
    "create_time": "2026-06-04T10:00:00Z"
  }
  ```
- **错误响应**：`404 Not Found`（短码不存在）

*注意：本接口为可选功能，当前版本暂不实现，仅作为后续扩展设计。*
